# 점심/저녁 메뉴 추천 알림봇 기획안

> 사내(여의도) 망분리 PC용 데스크톱 알림봇 · 무(無)DB · txt 파일 기반 · 단일 exe 배포
> 작성일: 2026-06-10

---

## 1. 배경 및 목적

여의도 사무실 직원들이 매일 반복하는 **"오늘 뭐 먹지?"** 의사결정 피로를 줄이는 것이 목적이다. 점심·저녁·회식 메뉴와 장소를 봇이 대신 골라주거나, 랜덤 게임으로 즐겁게 결정하도록 돕는다.

핵심 제약은 **사내 업무망(망분리)** 환경이라는 점이다. 외부 인터넷 접속이 불가능하므로 실시간 지도·날씨·맛집 API를 쓸 수 없고, 별도 DB 서버도 두지 않는다. 따라서 모든 데이터는 **지정된 경로의 txt 파일**로 관리하고, 프로그램은 **설치 없이 더블클릭으로 실행되는 단일 exe**로 배포한다.

### 설계 원칙
- **오프라인 우선(Offline-first):** 네트워크 없이 100% 동작.
- **무설치·무권한:** 관리자 권한 없이 exe 더블클릭만으로 실행.
- **데이터=txt:** 누구나 메모장으로 열어 메뉴/맛집을 추가·수정 가능.
- **가벼움:** 실행 즉시 뜨고, 메모리 점유 최소.
- **재미:** 단순 추천을 넘어 랜덤 게임 요소로 참여 유도.

## 2. 핵심 사용자 시나리오

1. **점심 직전(11:30)** 데스크톱 알림이 뜬다 → "오늘 점심 뭐 드실래요?" → 클릭하면 봇 실행.
2. 사용자가 **룰렛**을 돌려 3초 만에 메뉴 결정.
3. 팀 회식 때 인원수·예산을 입력하면 **회식장소 추천**.
4. 비 오는 날 **"가까운 곳만"** 필터로 우산 없이 갈 수 있는 식당만 추천.
5. 맛집을 새로 발견하면 **직원 추천**으로 등록 → 리스트에 반영.

## 3. 기능 명세

### 3.1 사용자가 제안한 기능 (필수)

| # | 기능 | 설명 |
|---|------|------|
| F1 | **랜덤 게임** | 룰렛 / 사다리타기 등으로 메뉴·식당을 무작위 선택. 시각적 재미 요소. |
| F2 | **회식장소 추천** | 인원수, 예산, 분위기(조용함/단체석) 조건으로 회식 가능한 식당 추천. |
| F3 | **날씨 연동** | "비/눈" 상태일 때 가까운(도보 N분 이내) 식당에 가중치. 망분리이므로 날씨는 **수동 선택 토글** 또는 사내 인트라넷 날씨 파일 연동(아래 8장). |
| F4 | **직원 추천 등록** | 직원이 식당/메뉴를 추천하면 리스트에 추가(`suggestions.txt`에 append). |
| F5 | **txt 데이터 관리** | 지정 경로의 txt 파일을 읽어 메뉴/맛집 목록 구성. DB 미사용. |

벤치마킹한 점심룰렛·Restaurant Roulette·Lunch Picker 등에서 공통적으로 효과가 검증된 기능들이다.

| # | 기능 | 왜 필요한가 |
|---|------|------------|
| A1 | **최근 중복 방지(쿨다운)** | 최근 N일 안에 나온 메뉴는 제외/감점. "또 김치찌개?" 방지. 가장 많이 요청되는 기능. |

| A8 | **txt 자동 리로드** | 파일 수정 시간을 감지해 재실행 없이 최신 목록 반영. |

## 4. 기술 스택 비교 및 추천

망분리·무설치 exe·가벼움이라는 요건에서 세 가지 후보를 비교했다.

| 항목 | **Python + PyInstaller** | **C# / .NET (WinForms·WPF)** | **Electron (JS)** |
|------|--------------------------|------------------------------|-------------------|
| 개발 속도 | ★★★ 매우 빠름 | ★★ 보통 | ★★ 보통 |
| exe 용량 | 15~40MB | 5~70MB(자체포함) / 작게 가능 | 100MB+ |
| 실행 속도·메모리 | 가벼움 | 가장 가벼움(네이티브) | 무거움 |
| GUI | tkinter / PySide6 | WinForms·WPF (윈도우 네이티브, 미려) | 웹기술(가장 자유로움) |
| 사내 백신 신뢰도 | △ PyInstaller exe가 **오탐(false positive)** 종종 발생 | ◎ 네이티브 exe라 신뢰 높음 | ○ |
| 망분리 빌드 난이도 | 패키지 사전 반입 필요(휠 파일) | Visual Studio + SDK 반입 | npm 의존성 대량 반입(까다로움) |
| 데스크톱 알림 | plyer / win10toast | 기본 API(BalloonTip·Toast) 우수 | 기본 지원 |
| 학습 곡선 | 낮음 | 중간 | 중간 |

### 추천: **Python + PyInstaller** (1순위) — 단, 백신 오탐 검증 필수

- **이유:** 개발 속도가 가장 빠르고, txt 파싱·룰렛 로직·간단 GUI 구현이 쉬워 MVP를 빠르게 낼 수 있다. `--onefile` 옵션으로 단일 exe가 나오고 무설치 실행이 가능하다.
- **주의:** 사내 백신/EDR이 PyInstaller exe를 오탐할 수 있다. 배포 전 **사내 보안팀에 화이트리스트 등록** 또는 코드사이닝을 협의해야 한다. 이 검증이 어렵다면 2순위로 전환한다.

### 2순위: **C# / .NET 단일파일 게시(Self-contained single-file)**

- 사내 보안 신뢰도와 네이티브 성능·알림이 강점. 윈도우 전용 사내 PC라면 가장 "회사 친화적"인 선택. Visual Studio가 이미 깔려 있거나 .NET 경험이 있다면 적극 권장.

### 비권장: **Electron**

- 용량이 크고(100MB+) 메모리 점유가 높으며, 망분리 환경에서 npm 의존성 일괄 반입이 번거롭다. 화려한 UI가 꼭 필요한 게 아니라면 과한 선택.

> **결론:** 빠른 출시·간편 개발이면 **Python+PyInstaller**, 사내 보안 통과·장기 운영 안정성이 최우선이면 **C#/.NET**. 두 경우 모두 **데이터 계층(txt 스키마)은 동일**하게 설계하므로, 스택을 바꿔도 데이터는 재사용된다.

## 5. 데이터 설계 (txt 파일 스키마)

DB 대신 지정 폴더(예: `C:\LunchBot\data\`)에 아래 파일들을 둔다. 구분자는 **파이프(`|`)** 를 사용하고, `#`로 시작하는 줄은 주석/헤더로 무시한다. 인코딩은 **UTF-8 (BOM 포함)** 으로 통일해 메모장에서 한글이 깨지지 않게 한다.

### 5.1 `restaurants.txt` — 식당 마스터
```
# 이름 | 카테고리 | 도보분 | 가격대(1~3) | 인원수용 | 태그 | 비올때OK(Y/N)
한솥도시락 | 한식 | 3 | 1 | 4 | 도시락,혼밥,빠름 | Y
교다이야 | 일식 | 8 | 2 | 6 | 라멘,줄있음 | N
백제정육식당 | 한식 | 12 | 3 | 30 | 회식,단체석,고기 | N
```
- **도보분**: 거리 기반 필터(비 올 때 가까운 곳)에 사용.
- **인원수용**: 회식장소 추천 시 인원수와 비교.
- **태그**: 회식/혼밥/빠름/단체석 등 자유 키워드.
- **비올때OK**: 실내 통로 연결·가까움 등으로 우천 시 추천 가능 여부.

### 5.2 `menus.txt` — 메뉴 후보(식당과 무관한 "뭐 먹지" 룰렛용)
```
# 메뉴 | 카테고리 | 맵기(0~3)
김치찌개 | 한식 | 2
마라탕 | 중식 | 3
돈카츠 | 일식 | 0
```

### 5.3 `suggestions.txt` — 직원 추천(앱에서 append)
```
# 등록일 | 등록자 | 이름 | 카테고리 | 한줄평
2026-06-10 | 김OO | 신상버거 | 양식 | 패티 두툼 가성비굿
```
- 봇은 시작 시 이 파일을 읽어 `restaurants.txt`/`menus.txt` 후보에 합류시킨다(또는 관리자가 검수 후 마스터로 승격).

### 5.4 `history.txt` — 선택 이력(통계·중복방지)
```
# 날짜 | 모드 | 선택결과
2026-06-09 | 점심 | 한솥도시락
2026-06-10 | 회식 | 백제정육식당
```

### 5.5 `config.txt` — 설정
```
# key = value
data_path = C:\LunchBot\data
notify_lunch = 11:30
notify_dinner = 17:30
cooldown_days = 3
default_walk_limit = 10
```

> **장점:** 비개발자도 메모장으로 맛집을 추가/수정 가능하고, 폴더째 백업/공유하면 데이터 이전이 끝난다. **주의:** 동시 쓰기 충돌을 막기 위해 append는 파일 락 또는 사용자별 임시파일 후 병합 방식을 쓴다(공유 폴더 배포 시).

## 6. 추천 알고리즘 (간단·설명가능)

1. **후보 수집:** 모드(점심/저녁/회식)에 맞는 풀 로드 → 직원 추천 합류.
2. **필터:** 카테고리, 가격대, 맵기, (회식 시)인원수용, (우천 시)`비올때OK=Y` & 도보 ≤ 제한.
3. **중복 감점:** `history.txt`에서 최근 `cooldown_days`일 내 나온 항목은 제외 또는 가중치↓.
4. **가중 랜덤:** 남은 후보에 가중치를 두고 무작위 추출(즐겨찾기 +, 최근선택 −).
5. **표현:** 룰렛/사다리 애니메이션으로 결과 공개 → `history.txt`에 기록.

가중 랜덤은 "완전 무작위의 재미"와 "합리적 추천"의 균형을 준다. 로직이 단순해 오프라인·저사양에서도 즉시 동작한다.

## 7. UI/화면 구성(안)

- **메인:** 큰 버튼 3개 — [점심] [저녁] [회식], 상단에 날씨 토글(☀️/🌧️), 카테고리 칩.
- **결과:** 룰렛 위주 화면 → 당첨 식당/메뉴 카드(도보분·태그·한줄평) → [다시] [복사] [길찾기 메모].
- **추천 등록:** 작은 폼(이름·카테고리·한줄평) → 저장 시 `suggestions.txt` append.
- **이력/통계:** 이번 달 캘린더 + 최다 선택 TOP5.
- **설정:** 데이터 경로, 알림 시간, 쿨다운 일수.

가볍고 직관적인 단일 창(탭) 구성. 트레이 아이콘 상주로 알림 기능 유지.

## 7-A. 비주얼 아이덴티티 (★ 핵심)

기능 자체는 시중 서비스와 크게 다르지 않으므로, **차별화 포인트는 "비주얼"** 에 둔다. 회사 타이틀인 **"JB"** 를 메인 키워드로 삼아 귀여운 **픽셀아트** 스타일로 통일한다. 별도의 이미지 외주 없이도 코드/도트로 제작 가능하고, 망분리 환경에서 외부 폰트·이미지 의존 없이 PNG 스프라이트로 내장할 수 있어 적합하다.

### 컨셉
- **로고:** 픽셀 블록으로 쌓은 굵은 **"JB"** (토마토색) — 회사 타이틀을 전면에.
- **마스코트:** **"밥봇 JB"** — 김(노리)을 두른 귀여운 주먹밥(오니기리) 캐릭터. 표정으로 상황 표현.
- **표정 변형 4종:** `기본` / `배고픔(입 벌림)` / `고민중(룰렛 돌 때, 눈 ×)` / `당첨!(웃는 눈)`.

### 컬러 팔레트
| 용도 | HEX |
|------|-----|
| 메인(JB·강조) — 토마토 | `#E8552E` (그림자 `#A8331A`) |
| 배경 — 크림 | `#FFF6E0` |
| 김(노리) — 딥그린 | `#33402F` |
| 쌀밥 | `#FFFFFF` (외곽 `#D6CDB8`) |
| 볼터치 | `#FF9FB0` |
| 포인트(반짝) | `#FFD166` |

### 픽셀 규격
- 도트 그리드 기준: 아이콘 **16×16 / 32×32**, 마스코트 **13×12 셀** 기반.
- 모든 에셋은 **PNG(투명 배경)** + 정수배 확대(`Nearest Neighbor`)로 선명도 유지.
- UI에서 확대 시 `image-rendering: pixelated`(또는 각 플랫폼 대응)로 흐려짐 방지.

### 에셋 적용처
- **앱 아이콘 / exe 아이콘**(256, 32): 바탕화면·작업표시줄.
- **트레이 아이콘**(32): 상주 알림.
- **알림 토스트**: "배고픔" 표정 + "오늘 뭐 먹지?".
- **룰렛 진행 화면**: "고민중" 표정, 결과 공개 시 "당첨!" 표정 + 반짝 효과.
- **빈 상태(Empty state)**: 데이터가 없을 때 마스코트가 "메뉴를 추가해 주세요" 안내.

> 제공 에셋: `jb_app_icon_256.png`, `jb_tray_icon_32.png`, `jb_mascot.png`(+happy/hungry/dizzy), `jb_expressions.png`. 생성 스크립트(`make_assets.py`)도 함께 제공하여 색상·표정을 코드로 재생성·수정 가능.

## 7-B. 추가 보완 사항 (검토 결과)

비주얼 외에 기존 기획에서 빠져 있던 항목들을 보완한다.

- **사운드(선택):** 룰렛 회전음·당첨음 등 짧은 효과음을 내장(wav)하고 on/off 토글 제공. 사무실 환경이므로 **기본은 무음**, 볼륨 작게.
- **첫 실행 온보딩:** 최초 실행 시 데이터 폴더가 없으면 **샘플 `restaurants.txt`/`menus.txt`를 자동 생성**하고 3컷 튜토리얼 표시. "데이터 없음=빈 화면" 방지.
- **에러/예외 처리:** txt 인코딩 오류, 빈 파일, 잘못된 줄은 **건너뛰고 경고 로그**(`error.log`)만 남긴 뒤 정상 동작. 프로그램이 죽지 않게.
- **개인정보 최소화:** 직원 추천 등록 시 실명 대신 **닉네임/부서**만, 또는 익명 옵션. 사내 개인정보 지침 준수.
- **운영/배포 관리:** `config.txt`에 버전 표기, 갱신은 exe 교체 방식. 데이터(`data\`)는 공유 폴더, 실행파일은 각 PC 권장(중복 쓰기 분리).
- **도움말/정보:** 사용법·문의처(담당자)·버전 표시하는 간단 도움말 화면.
- **성과지표(KPI):** `history.txt` 기반으로 일/주 사용 횟수, 추천 수락률(재추첨 비율), 인기 메뉴 TOP을 월간 리뷰. 사용 저조 시 개선 근거로 활용.
- **접근성:** 색맹 고려해 색상만으로 정보 구분 금지(아이콘·텍스트 병기), 최소 글자 크기 확보.

## 9. 참고한 외부 스킬/저장소 검토 결과

요청하신 5개 저장소를 이 프로젝트 관점에서 검토했다. **모두 Cowork 마켓플레이스에는 없으며**, 타깃 PC가 망분리라 그 PC에 직접 설치/사용하는 것은 불가하다. 단, **개발 워크스테이션에서 기획·코딩 보조용**으로 참고할 가치는 아래와 같이 차등적이다.

| 저장소 | 이 프로젝트 적합성 | 활용 포인트 |
|--------|-------------------|------------|
| **agentmemory** | ◎ 개념 적합 | "추천 이력/직원 추천을 파일로 기억"하는 발상과 동일. txt 기반 메모리 스키마 설계에 참고. |
| **Superpowers** | ○ 보조 | 범용 스킬/워크플로 프레임워크. 개발 단계의 작업 분해·자동화 보조용. |
| **Karpathy Guidelines** | △ 낮음 | LLM/에이전트 설계 원칙. 이 봇은 오프라인 규칙기반이라 LLM을 안 넣으면 직접 적용성 낮음. |
| **claude-video** | ✕ 무관 | 영상 생성 도구. 본 프로젝트와 무관. |
| **Understand-Anything** | ✕ 무관 | 멀티모달 이해 도구. 텍스트/exe 봇과 무관. |

**대신 Cowork에서 바로 쓸 수 있는 대안:** `Productivity` 플러그인의 `memory-management` 스킬이 CLAUDE.md + `memory/` 디렉터리 기반 **파일 메모리**라, 위 agentmemory 발상을 오프라인 친화적으로 대체한다. 기획·작업관리(`task-management`)도 이 프로젝트 진행에 유용하다.

## 12. 기획 검토 보완 사항 (2026-06-10 리뷰 반영)

개발 착수 시점의 기획 리뷰에서 발견된 빈틈과 확정 사항. **전부 v0.1.0 구현에 반영 완료.**

### 12.1 논리적 구멍 보완
- **자동 시작:** A7 알림은 앱이 켜져 있어야 동작 → 설정 화면에 "Windows 시작 시 자동 실행" 토글 추가(관리자 권한 불필요, `shell:startup` 바로가기). 온보딩에서 권유.
- **중복 실행 방지:** mutex 기반 단일 인스턴스 + 로컬 소켓 IPC. 트레이 상주 중 exe를 다시 실행하면 기존 창이 앞으로 올라온다. 토스트 클릭도 같은 경로로 기존 창을 띄운다.
- **경로 해석 규칙:** `config.txt`는 exe 폴더 고정(읽기 전용이면 `%LOCALAPPDATA%\LunchBot` 폴백). `data_path` 상대경로는 exe 기준으로 해석.

### 12.2 데이터 정책 확정
- **개인/공유 분리:** `history/favorites/exclude`는 개인 데이터(쿨다운·즐겨찾기는 개인의 것), `restaurants/menus/suggestions/weather`는 공유 데이터. `config.txt`의 `personal_path`로 분리 배치 가능(기본은 동일 폴더).
- **즐겨찾기/제외 스키마:** `favorites.txt`/`exclude.txt`(한 줄에 이름 하나). 즐겨찾기 = 가중치 ×1.6, 제외 = 무조건 필터 아웃.
- **weather.txt 형식:** `날짜 | 상태(맑음/비/눈)` 또는 상태 단독. 날짜가 오늘이 아니면 무시(어제의 '비' 방지). UI 수동 토글이 항상 우선.
- **가격대 정의:** 1 = ~1만원, 2 = 1~2만원, 3 = 2만원~.
- **history 유형 분리:** 모드 컬럼으로 메뉴/식당 이력을 구분해 쿨다운 매칭("김치찌개" 메뉴 선택이 "김치찌개집" 식당을 감점하지 않음).

### 12.3 에지 케이스 정책
- **후보 0개:** 단계적 완화(도보 제한 2배 → 날씨 필터 해제 → 카테고리 해제) 후 적용 내용을 결과 카드에 안내. 그래도 0개면 배고픈 마스코트의 빈 상태 화면.
- **전원 쿨다운:** 쿨다운은 제외가 아닌 감점(×0.15)이라 후보가 고갈되지 않는다.
- **공유 폴더 단절/파일 잠김:** 읽기 실패 시 마지막 성공 캐시로 동작, append는 5회 재시도.
- **재추첨 이력 오염:** [다시]는 기록하지 않고 [이걸로 결정]을 눌러야 history에 기록 — 수락률 KPI 왜곡 방지.
- **룰렛 후보 과다:** 8칸 초과 시 가중 선샘플링으로 8개만 표시("후보 N개 중 8개 표시" 안내).

### 12.4 안정성 정책
- **알림 스케줄:** 1회 타이머가 아닌 30초 폴링 — 절전 복귀에도 동작. 시각을 놓치면 `notify_catchup_min`(기본 60분) 이내에 한해 늦게라도 발송.
- **스레딩 규칙:** 트레이/IPC/스케줄러 스레드 → UI 전달은 큐 + `after()` 폴링만 사용. 창 X 버튼 = 트레이로 숨김(첫 회 안내 토스트), 종료는 트레이 메뉴.
- **회식 입력 검증:** 인원은 스피너(2~50), 예산은 구간 드롭다운 — 자유 입력 제거.
- **사운드:** v0.1.0 미구현(기본 무음 정책 유지, 3차 이후 검토).
