# 밥봇 JB — UI 전면 개편 기획안 v2.0

> 디자인 컨셉: **"Midnight Bento × Pixel Soul"** · 작성일: 2026-06-10
> 본 문서는 [점심메뉴_추천_알림봇_기획안.md](점심메뉴_추천_알림봇_기획안.md)의 7장(UI)·7-A장(비주얼)을 **대체**한다. 기능 명세(3장)·데이터(5장)·알고리즘(6장)·망분리 대응(8장)은 v1 그대로 유효하다.

## 1. 개편 배경

v0.1.0의 tkinter/ttk UI는 기능적으로는 완성됐으나 시각적으로 "80~90년대 윈도우 프로그램" 인상을 준다. 원인은 스타일이 아니라 **프레임워크 한계**다: 사각 위젯, 시스템 회색 테마, 그림자·라운드·호버 불가. 사내 확산의 관건이 "재미·매력"인 제품 특성상, 트렌드에 맞는 전면 개편을 진행한다.

**기술 전환: tkinter/ttk → CustomTkinter** (순수 Python — 망분리 100% 안전, 단일 exe 유지, 라운드·다크·호버 내장). 로직 계층은 무수정, UI 계층만 교체한다.

## 2. 벤치마킹 조사 결과

### 2.1 글로벌 데스크톱 — "Linear 디자인" 세대가 표준
Linear·Raycast·Vercel·Arc 등 현 세대 데스크톱 앱의 공통 문법:

| 원칙 | 내용 | 밥봇 적용 |
|------|------|----------|
| 다크 퍼스트 | 다크가 "진짜" 버전, 라이트는 파생 | 다크 단일 모드로 설계 |
| 그림자 금지 | 다크에선 그림자가 안 보임 → **1px 보더 + 휘도 단계**로 깊이 | 표면 5단계 휘도 토큰 |
| 색의 절제 | 모노크롬 + 포인트 1~2색만 | 토마토(#E8552E) 단일 포인트 |
| 볼드 타이포 | 큰 헤드라인 + 충분한 여백 | DISPLAY 20pt bold 위계 |
| 절제된 글래스 | 블러는 오버레이에만 살짝 | 토마토 틴트 보더로 모사 |

### 2.2 한국 앱 (토스·배민 계열)
- 디자인 토큰을 코드와 1:1 연결하는 시스템 운영(TDS) → 본 문서 3장의 토큰표가 곧 `theme.py`
- 2025 국내 트렌드: 다크모드의 전략화, 벤토 그리드, **아날로그·레트로 감성의 부활** → **JB 픽셀 오니기리는 버릴 자산이 아니라 증폭할 자산**. 다크 배경 위 네온 글로우 픽셀로 재해석한다.

### 2.3 스핀휠/룰렛 UI 문법 (Dribbble·게임 UI)
- 휠은 "히어로 그래픽": 외곽 장식 링, 발광 포인터, 회전 중 틱 피드백
- **당첨 순간의 축하 연출(컨페티·펄스)이 표준** — 보상 감각이 재사용률을 만든다
- 칸이 많으면 줄여서 가독성 우선 (기존 8칸 선샘플링 유지)

### 2.4 모션 표준 (Material 3 모션 토큰 준용)
- 소형(호버·칩 토글) 50~150ms, 중형(카드 등장·페이드) 200~400ms
- 히어로 모먼트(룰렛 스핀·당첨)는 emphasized 이징으로 길고 드라마틱하게
- 모든 모션은 목적이 있어야 하며 UI 전환은 300ms를 넘기지 않는다

## 3. 디자인 시스템

### 3.1 철학 — "Midnight Bento × Pixel Soul"
Linear식 절제된 다크 + 벤토 그리드 골격 위에, JB 픽셀 마스코트와 토마토 네온이 **유일한 감성 포인트**로 발광한다. *"기능은 미니멀하게, 재미는 픽셀로."*

### 3.3 타이포·형태·간격
- 폰트: 맑은 고딕 (전 Windows 기본 — 번들 불요). 크기 위계: 11(본문) / 13(강조) / 15(섹션) / **20 bold(DISPLAY — 결과·버튼)**
- 카드 radius **14**, 칩·주요 버튼 radius **999(필 형태)**, 입력 radius 8
- 간격: 8pt 그리드 (8/16/24), 카드 내부 패딩 16

### 3.4 모션 토큰

| 토큰 | 시간 | 이징 | 적용 |
|------|------|------|------|
| micro | 120ms | ease-out | 칩 토글 피드백, 텍스트 페이드 |
| small | 180ms | ease-out cubic | 사이드바 인디케이터 슬라이드 |
| medium | 250ms | ease-out cubic | 결과 카드 슬라이드 인, 카운트업(300ms) |
| hero-spin | 2.6s | 와인드업 150ms → **quintic ease-out** | 룰렛 본회전 |
| hero-win | 1.2s | 물리(중력+드리프트) | 컨페티 50파티클 + 칸 펄스 3회 |
| idle | 4s 주기 | — | 마스코트 눈 깜빡임 |

## 4. 그래픽 강화 계획 (이번 개편의 핵심)

1. **다크 네온 마스코트** — 픽셀 외곽에 2단계 토마토 글로우 링(알파 150/60)을 자동 생성(`add_glow()`). 다크 전용 팔레트: 쌀밥 외곽선 `#EDE6D4`(밝힘), 김 `#3E4D39`(배경에 묻힘 방지). 표정 4종 + 로고의 `*_dark` 변형 생성.
2. **살아있는 마스코트** — 눈 깜빡임 프레임 추가, 사이드바 마스코트가 4초마다 깜빡인다.
3. **히어로 룰렛** — 휠 외곽 네온 글로우 동심원 링(회전 중 맥동), 와인드업 후 quintic 감속, 포인터 아래 칸 실시간 하이라이트(틱), 허브에 미니 마스코트, **당첨 시 칸 3회 펄스 + 컨페티**.
4. **벤토 통계 그래픽** — 이력 화면 숫자 카운트업 + 순위·득표 토마토 수평 바.
5. **시간대 인사 카피** — 홈 헤더에 시간 연동 문구("11:20 — 슬슬 배고프죠? 🍙").

## 5. 화면별 기획

### 5.1 공통 레이아웃 — 사이드바 + 콘텐츠 (창 1000×680)

```
┌──────────┬────────────────────────────────────────────────┐
│ [JB 네온] │ 11:20 — 슬슬 배고프죠? 🍙          날씨 [☀|🌧|❄] │
│ 밥봇 JB   │ ╭─조건 벤토(320px)──╮ ╭─룰렛 카드──────────────╮ │
│          │ │ 모드               │ │      ▼ 네온 포인터     │ │
│ ▌🎰 추천  │ │ [점심|저녁|회식|메뉴]│ │   ◎ 글로우 링 휠      │ │
│  🗳 투표  │ │ 카테고리            │ │   (중앙 미니 마스코트)  │ │
│  ✍ 등록  │ │ (✓한식)(중식)(일식) │ │                      │ │
│  📊 이력  │ │ ⌕ 검색             │ │  고민중… (후보 12개)   │ │
│  ⚙ 설정  │ │ 회식: 인원[−8+] 예산▾│ ╰──────────────────────╯ │
│  ❓ 도움말 │ │ ▶ 룰렛 돌리기!      │ ╭─결과(슬라이드 인 250ms)─╮│
│          │ ╰───────────────────╯ │ 🎉 김밥천국             ││
│ (마스코트  │                       │ 분식 · 도보 4분 · 가격 1 ││
│  깜빡임)  │                       │ [✅결정][🔄][📋][⭐][🚫] ││
│  v2.0    │                       ╰────────────────────────╯│
└──────────┴────────────────────────────────────────────────┘
```
- 사이드바: 활성 메뉴 좌측 4px 토마토 인디케이터가 슬라이드(180ms) 이동. 하단에 깜빡이는 마스코트 + 버전.
- 회식 조건은 모드=회식일 때만 표시. 결과 카드의 즐겨찾기/제외 버튼 동작은 v1 로직 그대로.

### 5.2 투표
후보마다 개별 카드(hover lift). 득표 비례 토마토 바, [+1]·[거부]. 거부 시 카드 회색 + 이름 취소선(색+형태 이중 신호). [🏆 결정!] → 동률 랜덤 → 기록.

### 5.3 등록
중앙 1열 폼 카드(최대폭 480) — placeholder 라벨, 성공 시 카드 하단 "✅ 등록 완료" SUCCESS 표시.

### 5.4 이력 — 벤토 통계
```
┌이번 달┐┌점심┐┌저녁┐┌회식┐      ← 카운트업 애니메이션
│ 14회  ││ 9회 ││ 4회 ││ 1회 │
└──────┘└────┘└────┘└────┘
┌TOP 5 ─────────────┐┌최근 20건────────────┐
│🥇 김밥천국 5회 ████ ││06-10 [점심] 김밥천국 │
└───────────────────┘└─────────────────────┘
```

### 5.5 설정 / 도움말
설정: 경로/알림/동작/제외목록 **4개 카드**로 분할, 제외목록은 행 더블클릭 해제 유지. 도움말: 1카드 Textbox, 섹션 제목 토마토 볼드.

## 6. 컴포넌트 인벤토리 (`ui/widgets.py` 신규)

| 컴포넌트 | 구성 | 비고 |
|----------|------|------|
| `Card` | CTkFrame(radius 14, border 1) + 타이틀 라벨, hover 시 `BORDER_GLOW` | 벤토 기본 단위 |
| `Chip` | 필 토글 버튼 — 선택 시 `TOMATO_SOFT` 배경 + "✓ " 접두 | 카테고리 필터 |
| `Stepper` | `[−] n [+]` (CTk에 Spinbox 없음) | 회식 인원·투표 후보 수 |
| `ListPanel` | CTkScrollableFrame + 행 hover/클릭/더블클릭 | 이력·제외목록 |
| `animate()` | after 기반 프레임 보간 공통 헬퍼 | 슬라이드·페이드·카운트업 공용 |

위젯 매핑: LabelFrame→Card, Radiobutton→CTkSegmentedButton, Checkbutton→Chip, Spinbox→Stepper, Listbox→ListPanel, Combobox→CTkOptionMenu, Text→CTkTextbox.

## 7. 접근성·품질 기준

- 텍스트 명도 대비 4.5:1 이상 (TEXT_MID 이상을 본문에 사용)
- 색상 단독 정보 전달 금지 — 선택 칩 "✓", 거부 취소선, 상태 아이콘 병기
- 모든 모션은 기능을 막지 않음(논블로킹), 룰렛 회전 중 버튼만 비활성
- 픽셀아트는 NEAREST 정수배 확대만 사용 (뭉개짐 금지)
- HighDPI 100/125/150% 검수

## 8. 구현 범위·일정

| 단계 | 내용 |
|------|------|
| R1 | 디자인 토큰·에셋(글로우 마스코트)·공통 위젯 |

| R3 | 투표/등록/이력/설정/도움말 전환 |
| R4 | 검증(테스트·스크린샷·DPI)·빌드·배포 |
